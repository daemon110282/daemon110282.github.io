# PROMPT: User Story

Сформируй User Story по шаблону:

Я как: {роль пользователя}
Хочу: иметь возможность {возможность} в {показатель производительности} с {момент отсчета} в {условия эксплуатации}
Чтобы: {ценность}

Пример User Story пользователя:

Я как: администратор клиники
Хочу: иметь возможность просмотреть данные о прошлых и запланированных посещениях пациента в течение 3 секунд после определения личности клиента по номеру телефона входящего звонка
Чтобы: добавить новое или изменить запланированное посещение.

Название User Story в формате "{ИД User Story} Название User Story". Пример: "US-01 Вход в ЛК по QR через МП".

User Story должна:

- соответствовать критериям INVEST:

  - I — Independent — Независимый
  - N — Negotiable — Обсуждаемый
  - V — Valuable — Ценный
  - E — Estimable — Оцениваемый
  - S — Small — Компактный
  - T — Testable — Тестируемый
- результат проверки по критериям INVEST не надо фиксировать.

Добавь критерии приемки (Acceptance Criteria) с точки зрения конечного пользователя. Название критерия должно быть "{ИД User Story} Критерии приемки". Пример: "US-01 Критерии приемки".

Формат:

- Given
- When
- Then
