# Локальная LLM

## Зачем

- использовать локальные модели
- Плюсы
  - экономить на API
  - обеспечить конфиденциальность данных
  - быстро получать ответы без сетевых задержек
- Минусы
  - ограниченные ресурсы по сравнению с облачными решениями
  - необходимость настройки и обслуживания моделей
  - дорогое железо для больших моделей

## Технологии

- [Ollama](https://skillbox.ru/media/code/ollama/)
  - стоит использовать, когда нужен простой и быстрый запуск локальных LLM без глубокого погружения в настройки. Работает на llama.cpp.
  - [llama.cpp + QWEN + подбор под оборудование](https://habr.com/ru/companies/first/articles/1017822/)
    - предпочтителен, когда важны максимальная производительность, минимальные накладные расходы, тонкая настройка параметров инференса, нестандартные сценарии развёртывания, интеграция на низком уровне или работа в ограниченных средах (например, собственные серверы, встроенные системы, офлайн-инсталляции без дополнительных сервисов)
      - модель в __формате GGUF__ - Это специальный формат с поддержкой __квантизации__, позволяющей уменьшить размер модели и снизить требования к памяти
- LM Studio
- VS Code
