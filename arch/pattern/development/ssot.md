# Принцип Single Source of Truth (SSOT)

это принцип, согласно которому вся компания использует один источник данных для ключевых бизнес‑сущностей: клиентов, продуктов, заказов, метрик.
Вместо множества разных определений, таблиц и формул у каждого отдела появляется [единый стандарт интерпретации информации](https://habr.com/ru/companies/modusbi/articles/952072/).

## Зачем

Решение проблем рассинхронизации состояния бизнес-сущностей, например:
ЭДО меняет состав пакета, ЛК уже имеет свою копию (или проекцию состояния), но:

- нет механизма синхронизации или инвалидации старого состояния,
- нет события типа "пакет 1 устарел / аннулирован".

В итоге существуют две конкурирующие истины:
  
- Пакет 1 (в ЛК, уже подписан)
- Пакет 2 (в ЭДО, актуальный)
